** Methodological Findings
1. *glob_huber_outlier = glob_huber*: After Huber fit, no residuals exceed
   zscore threshold — outlier removal adds nothing on top of Huber loss.
2. *glob_irls = glob_iterative*: Both reweight branches execute the same
   code (set y_err = mean |residual|). Should differentiate or merge.
3. *Huber stderr is BROKEN* without physics-based y_err: produces absurd
   stderr (1e5-1e7) for 1-label data AND uniform/no-error data. The Huber
   loss in lmfit does not properly propagate errors when starting weights
   are uniform. With physics-based or SEM-based heteroscedastic y_err,
   Huber stderr is well-behaved (SE/SD = 0.69-0.93x).
4. *ODR consistently underestimates uncertainty*: SE/SD = 0.13-0.34x across
   all scenarios. Known issue: assumes orthogonal errors.
5. *Error ratio does NOT change method rankings*: glob_huber > glob_ls >
   outlier2 > IRLS/iterative > ODR holds at all error ratios (0.1, 0.5, 1.0).
6. *2 outliers*: glob_huber handles 2 outliers as well as 1 (91.8% cov).
   glob_ls drops slightly (89.2%) but remains robust. IRLS drops badly (76.6%).
7. *SEM-based weights help Huber* (96.8% cov!) but hurt glob_ls (84.2%).
   The scalar-per-label SEM assigns equal weight within a label, which
   oversmooths heteroscedastic per-point errors but properly captures the
   y1-vs-y2 ratio. Huber loss can exploit this to downweight outliers.
8. *Uniform weights*: glob_irls slightly beats glob_ls (88.6% vs 87.6%).
   Both are reasonable fallbacks when physics errors are unavailable.

** Key Conclusions
1. *Best 2-label with physics y_err*: glob_huber (91-92% cov, robust to 1-2
   outliers). The physics error model naturally handles heteroscedasticity;
   Huber loss adds outlier resistance on top.
2. *Best 1-label*: glob_ls (88-89% cov). Huber stderr is unreliable for
   1-label; glob_ls is safe and well-calibrated (SE/SD = 0.92x).
3. *Best fallback (no physics y_err)*: glob_ls or glob_irls (87-89% cov).
   Huber stderr is broken with uniform weights.
4. *IRLS/iterative always hurts*: reweighting amplifies outlier bias and
   drops coverage 5-15% vs glob_ls. Never the right choice.
5. *outlier2 outperformed by glob_ls*: deprecated, should be retired.
6. *No method reaches 95% CI*: best is 96.8% (Huber+SEM weights) but
   the standard scenario tops out at 91.8%. The 7-point titration with
   3+ params per label inherently limits precision.

3. *multi-noise is best for L4* (std 0.355, −6% vs huber-lb2): the per-step
   noise term accommodates slightly more pH variability between wells.
4. *multi-noise-xrw does NOT help*: for L2 it is WORSE than single (std 0.293
   vs 0.289); for L4 it equals single (both 0.381). The per-well pH random
   walk adds unnecessary uncertainty when pH is well-calibrated by additions.
   xrw may be appropriate only when individual well pH was NOT measured.
5. *Sampler viability*: only the default pytensor/CPU sampler is reliable for
   all modes. nutpie hangs on hierarchical (multi+), numpyro is broken (JAX
   version mismatch), blackjax works for single only (CPU fallback after OOM).

* TODO Heuristic y_err correction via proportional noise α
** Background
The current heuristic y_err = sqrt(signal + bg_err²) captures only readout
noise and Poisson noise. The true variance model revealed by multi-noise MCMC is:
  Var(y) = σ_read² + gain·μ + α²·μ²
where the *proportional* term α²·μ² is DOMINANT at typical fluorescence signal
levels (200-1000 RFU).  This means high-signal y1 wells are systematically
under-weighted (treated as more reliable than they really are).

** Can α be estimated from input data statistics?
Three approaches were evaluated, all failing for different reasons:
1. CV of replicate wells: dominated by biological/concentration variance, not
   instrument noise.  Estimate: ~0.45 vs MCMC ~0.06 — completely wrong.
2. Median(|residual|/signal) from single-well fits: S0/S1 absorb proportional
   noise into plateau parameters.  Result: 0.43 vs MCMC 0.057 for L2 y1.
3. OLS regression Var(r) ~ σ²+gain·s+α²·s²: works for high-SNR labels where
   α dominates (L4 y1: 0.104 vs MCMC 0.102; L4 y2: 0.033 vs MCMC 0.046)
   but fails when σ_read is large (L2 y1: α→0 from regression vs MCMC 0.057).
   Only 7 pH steps + free S0/S1 make the 3-parameter decomposition ill-conditioned.

*Conclusion*: α cannot be reliably estimated from raw data statistics.
MCMC multi-noise is the only reliable estimator. α values from MCMC should
be stored and reused as empirical priors for the heuristic y_err correction.
