** Methodological Findings
1. *glob_huber_outlier ≡ glob_huber*: After Huber fit, no residuals exceed
   zscore threshold — outlier removal adds nothing on top of Huber loss.
2. *glob_irls ≡ glob_iterative*: Both reweight branches execute the same
   code (set y_err = mean |residual|). Should differentiate or merge.
3. *Huber stderr unreliable* for 1-label at K=6.0: lmfit Huber loss sometimes
   returns absurd stderr (1e7). For 2-label, stderr is well-behaved.
4. *ODR consistently underestimates uncertainty*: SE/SD ≈ 0.29–0.34x → only
   ~40% coverage. Known issue: assumes orthogonal errors.

** Key Conclusions
1. *Best overall*: glob_ls (standard WLS) — best SE/SD calibration (0.90x),
   lowest RMSE, 90% coverage. The physics error model (sqrt(signal + buffer^2))
   already downweights noisy y1 points proportionally.
2. *Best with outliers*: glob_huber — gains ~2% coverage over glob_ls when
   outliers present, with reasonable SE/SD (0.79x).
3. *IRLS/iterative hurt*: reweighting amplifies bias (-0.095 vs -0.049 for
   glob_ls) and drops coverage by ~12% in the outlier scenario.
4. *outlier2 underperforms*: coverage ~84% (clean) and ~85% (outlier), worse
   than glob_ls in both cases. Inherited from pre-refactor implementation.
5. *No method reaches 95% CI*: best is 91.7% (Huber, 2-label clean). The
   7-point titration with 3+ params per label inherently limits precision.
6. *SE/SD < 1 everywhere*: all methods underestimate stderr by 10–30%,
   confirming the need for scale_covar=True (sqrt(reduced_chi_sq) inflation).

** Updated Recommendation
- *PRIMARY*: fit_binding_glob(ds) — standard WLS with physics weights
  is the best default. scale_covar=True handles mild outliers.
- *ROBUST*: fit_binding_glob(ds, method="huber") — use when y1 outliers
  are suspected. Slightly inflated RMSE but better coverage.
- *AVOID*: reweight="irls"/"iterative" — amplifies outlier bias.
- *RETIRE*: outlier2() — no coverage advantage over glob_ls.
- *FIX*: ODR stderr (SE/SD 0.3x) before using for production.
